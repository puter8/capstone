# Pally 기능 명세서 (v1.0)

> 작성일: 2026-05-21  
> 데모 목표일: 2026-06-08  
> 범위: MVP (June 7 데모) + Post-MVP 전 기능 포함

---

## 서비스 개요

**서비스명:** Pally (CharaShift)  
**팀명:** 퓨터 (산학 트랙)  
**한 줄 정의:** 사용자의 영어 발화 스타일을 5개 축으로 분석하고, 그 결과에 따라 AI 캐릭터의 성격이 실시간으로 변하는 개인화 영어 회화 학습 서비스

---

## 목차

1. [로그인/회원가입] 소셜 계정 인증
2. [온보딩] 레벨·캐릭터·유저 이름 설정
3. [채팅 화면] 메인 대화 인터페이스
   - [발화 분석] CHARACTER MATRIX 5축 분석 엔진
   - [캐릭터 응답] AI 캐릭터 응답 생성
   - [AI 피드백] 회화 교정 및 피드백
   - [STT] 음성 입력
   - [TTS] 음성 출력
4. [캐릭터 시각화] Canvas2D Superformula 캐릭터
5. [5축 패널] 발화 스타일 수치 시각화
6. [세션 관리] 세션 상태 및 토큰 관리
7. [결제] 유료 플랜 및 토큰 구매
8. [대화 이력] 이전 대화 조회 *(Post-MVP)*
8. [ML 발화 분석기] 딥러닝 기반 분석 고도화 *(Post-MVP)*
9. [Slang RAG] Reddit 슬랭 파이프라인 *(Post-MVP)*
10. [안전장치] 과몰입 방지 및 윤리 설계 *(Post-MVP)*

## 1. 로그인/회원가입

### **[로그인/회원가입] 소셜 계정 인증**

- **영어 학습자로서, 별도 가입 절차 없이 소셜 계정으로 빠르게 로그인하고 싶다. 그래야 복잡한 폼 입력 없이 바로 서비스를 시작할 수 있기 때문이다.**

기능 상세 설명

1. **제공 로그인 방식:**
   - Google 로그인 ("Sign up with Google")
   - 카카오 로그인 ("카카오로 시작하기")
2. **화면 구성:**
   - 상단: 캐릭터(Pally) 이미지
   - 중단: 서비스 소개 문구
   - 하단: Google 버튼(흰 배경) + 카카오 버튼(노란 배경)
3. **인증 기술:** Supabase Auth OAuth
   - Google: `supabase.auth.signInWithOAuth({ provider: 'google' })`
   - 카카오: `supabase.auth.signInWithOAuth({ provider: 'kakao' })`
4. **최초 가입 vs 재로그인 분기:**
   - 최초 가입 → `/onboarding` 4단계 진행
   - 재로그인 → 이전 온보딩 완료 여부 확인 → `/chat` 직접 이동
5. **RLS(Row Level Security):** DB 레벨에서 본인 데이터만 접근 가능
6. **에러 처리:**
   - 소셜 로그인 취소 시: 로그인 화면 유지, 에러 토스트 없음
   - 네트워크 오류 시: "잠시 후 다시 시도해주세요" 안내

## 2. 온보딩

### **[온보딩] 4단계 설정 플로우**

- **영어 학습자로서, 처음 가입 시 내 영어 레벨과 AI 친구 이름을 설정하고 싶다. 그래야 나에게 맞는 난이도로, 내가 이름 붙인 캐릭터와 대화를 시작할 수 있기 때문이다.**

기능 상세 설명

전체 플로우: **로그인 → 온보딩 1 → 온보딩 2 → 온보딩 3 → 온보딩 4 → 채팅**

---

#### 온보딩 1 — 서비스 소개

- **기능:** 서비스 핵심 가치 소개 화면
- **표시 내용:** 캐릭터 애니메이션 + "대화할수록 나를 닮아가는 영어 친구"
- **CTA:** "계속하기 →" 버튼 → 온보딩 2로 이동
- **페이지 인디케이터:** 하단 점(●○○) 표시

---

#### 온보딩 2 — 영어 레벨 선택

- **기능:** 사용자의 현재 영어 수준을 선택하여 AI 응답 난이도를 조절
- **선택 항목:**

  | 레벨 | 설명 | 5축 초기 Formality 범위 |
  |------|------|------------------------|
  | A2 · 초급 | 기본 표현 중심, 짧고 단순한 대화 가능 | Formality: 30~40 |
  | B1 · 중급 | 일상적 주제 대화 가능, 약간의 오류 있음 | Formality: 40~60 |
  | B2 · 중상급 | 다양한 주제에 자연스럽게 대화 가능 | Formality: 50~70 |
  | C1 · 고급 | 유창하고 자발적인 고급 표현 사용 | Formality: 60~80 |

- **UI:** 라디오 버튼 리스트, 선택 항목은 주황 테두리로 강조
- **CTA:** "다음 →" 버튼 → 온보딩 3으로 이동

---

#### 온보딩 3 — AI 친구 이름 설정 (Pally 이름)

- **기능:** 사용자가 AI 캐릭터에게 직접 이름을 붙여 소속감과 친밀감 형성
- **표시 내용:**
  - 캐릭터 이미지 + "친구의 이름을 지어주세요"
  - 텍스트 입력창 (placeholder: e.g., "Zu-Zu")
  - 추천 이름 버튼: **Bobo / Mama / Coco / Nana** (탭하면 입력창 자동 채워짐)
- **유효성 검사:**
  - 1~10자 제한
  - 공백만 입력 불가
  - 특수문자 필터링
- **CTA:** "완료 →" 버튼 → 온보딩 4로 이동

---

#### 온보딩 4 — 유저 이름 설정

- **기능:** AI 캐릭터가 사용자를 부를 이름 설정. 대화 중 호칭에 사용
- **표시 내용:**
  - 캐릭터 이미지 + "유저 이름" 입력창
  - 추천 이름 버튼: **Bobo / Mama / Coco / Nana**
- **유효성 검사:** 온보딩 3과 동일
- **저장:** Supabase `user_profiles` 테이블에 저장:
  ```sql
  user_profiles (user_id, pally_name, user_name, english_level, created_at)
  ```
- **CTA:** "완료 →" → `/chat`으로 이동, 첫 대화 시작

---

- **콜드 스타트 대응:** 온보딩 직후 3~5턴은 EMA α를 0.7로 높게 설정하여 캐릭터 변화를 빠르게 체감하도록 함. 이후 정상 α(0.5)로 복귀

## 3. 채팅 화면

### **[채팅 화면] 메인 대화 인터페이스**

- **영어 학습자로서, AI 캐릭터와 자유롭게 영어로 대화하고 싶다. 그래야 실전 회화 감각을 키울 수 있기 때문이다.**

기능 상세 설명

1. URL: `/chat`
2. **3열 레이아웃** 구성:
   ```
   ┌──────────────┬────────────────────┬──────────────┐
   │  캐릭터 패널  │     채팅 영역       │  5축 패널    │
   │  (Canvas2D)  │  메시지 목록        │  Formality   │
   │              │  사용자 메시지(우)  │  Energy      │
   │  형태 실시간  │  AI 메시지(좌)     │  Intimacy    │
   │  변화        │  텍스트 입력창      │  Humor       │
   │              │  STT 버튼          │  Curiosity   │
   └──────────────┴────────────────────┴──────────────┘
   ```
3. **메시지 입력창:**
   - 영어 텍스트 타이핑 입력
   - Enter 키 또는 전송 버튼으로 전송
   - 마이크 버튼으로 STT 음성 입력 전환 가능
4. **메시지 목록:**
   - 사용자 발화: 오른쪽 정렬
   - AI 캐릭터 응답: 왼쪽 정렬 (스트리밍 타이핑 효과)
5. **AI임을 명시:** UI 상단에 "AI 캐릭터와 대화 중" 라벨 항상 노출

---

### **[발화 분석] CHARACTER MATRIX 5축 분석 엔진**

- **영어 학습자로서, 내 발화 스타일이 정량적으로 분석되길 원한다. 그래야 내가 어떻게 대화하는 사람인지 객관적으로 파악할 수 있기 때문이다.**

기능 상세 설명

1. **5축 입력 정의 (0~100 스케일):**

   | 축 | 설명 | 분석 기준 |
   |----|------|-----------|
   | Formality | 격식도 | 격식어/비격식어 키워드, 문장 완성도, 축약어 여부 |
   | Energy | 에너지 | 감탄사, 이모지, 반복 문자(!!, ??) 빈도 |
   | Intimacy | 친밀도 | 1인칭 대명사, 호칭, 슬랭 사용 여부 |
   | Humor | 유머 성향 | 밈 표현, 장난기 있는 어휘, 이모지 사용 |
   | Curiosity | 탐구 성향 | 의문문 빈도, 탐구형 어휘 ("why", "how", "wonder") |

2. **Rule-Based 분석기 (`ai/analyzer.py`):**
   - 키워드 딕셔너리 200개 (슬랭, 격식어, 감탄사, 밈)
   - 정규식 패턴 매칭 (축약어, 이모지, 문장 복잡도)
   - 각 축별 가중치 테이블 적용
   - 발화 1건 → 5개 축 점수(0~100) 반환

3. **EMA (지수이동평균) 누적 스무딩:**
   ```
   new_axes = α × raw_axes + (1 - α) × prev_axes
   ```
   - Intimacy: α=0.1 (천천히 누적 → 관계 형성의 시간성 반영)
   - 나머지 4축: α=0.5 (빠르게 반응)
   - 온보딩 직후 첫 3~5턴: α=0.7 (콜드 스타트 대응, 빠른 변화 체감)

4. **CHARACTER MATRIX 가중합 연산 (`ai/matrix_engine.py`):**
   ```
   params = W × [Formality, Energy, Intimacy, Humor, Curiosity]ᵀ + BIAS
   ```
   - W: 3×5 가중치 행렬
   - BIAS: [50, 20, 10]
   - 출력: tone_casual, energy_level, humor_level (각 0~100, clamp 처리)

5. **출력 캐릭터 파라미터 3축:**

   | 파라미터 | 설명 | 영향 |
   |----------|------|------|
   | tone_casual | 캐주얼 말투 정도 | GPT 응답 말투, Canvas2D 형태 |
   | energy_level | 응답 활발함 | GPT 응답 에너지, Canvas2D 크기 |
   | humor_level | 유머러스함 | GPT 응답 장난기, Canvas2D 색상 |

6. **에러 처리:**
   - 발화 분석기 실패 시: 이전 누적 점수 그대로 유지
   - 축 점수 범위 초과 시: `clamp(0, 100)` 자동 적용
   - Canvas NaN/undefined 시: 기본값 50 적용

7. **캐릭터 정체성 제어 로직 (Post-MVP 고도화):**
   - 기저 성격(Base Personality): 캐릭터 선택 시 고정, EMA로 변하지 않음
   - 관계 레이어(Relationship Layer): Intimacy 편차만 EMA로 업데이트
   - delta 범위 ±20으로 클램핑 → 캐릭터 정체성 붕괴 방지

### **[캐릭터 응답] AI 캐릭터 응답 생성**

- **영어 학습자로서, 내 발화 스타일에 반응하는 AI 응답을 받고 싶다. 그래야 획일적인 챗봇이 아닌 나만의 캐릭터와 대화하는 느낌을 받을 수 있기 때문이다.**

기능 상세 설명

1. **API 엔드포인트:** `POST /api/chat` (SSE 스트리밍)
2. **요청 형식:**
   ```json
   {
     "message": "yo what's up lol",
     "session_id": "UUID",
     "current_axes": { "Formality": 20, "Energy": 65, "Intimacy": 55, "Humor": 47, "Curiosity": 62 }
   }
   ```
3. **서버 처리 순서:**
   ```
   1. analyze_utterance(message)        → raw 5축 점수
   2. apply_ema(prev_axes, raw_axes)    → 누적 5축 점수
   3. compute_character(smoothed_axes)  → 캐릭터 파라미터
   4. build_system_prompt(params)       → GPT 지시문 구성
   5. GPT-4o 호출 (SSE 스트리밍)        → 캐릭터 응답
   6. Supabase 저장 (메시지 + 축 점수)
   7. 응답 반환: { reply, new_axes, character }
   ```
4. **시스템 프롬프트 구조:**
   ```
   You are an English conversation partner named Pally.
   Your personality adapts based on:
   - tone_casual: {value} → 0=very formal, 100=very casual/slangy
   - energy_level: {value} → 0=calm, 100=enthusiastic
   - humor_level: {value} → 0=serious, 100=playful/meme-friendly

   Respond in English only. 1-3 sentences max.
   [tone_casual > 60] Use contractions, casual words, light slang.
   [tone_casual < 30] Use formal vocabulary, polite phrasing.
   [energy_level > 60] Use exclamations, show excitement.
   [humor_level > 60] Light jokes, wordplay OK.
   Do NOT mention parameters. Do NOT claim to be human.
   ```
5. **응답 형식:**
   ```json
   {
     "reply": "omg heyyy!! what's the vibe today??",
     "new_axes": { "Formality": 18, "Energy": 65, "Intimacy": 55, "Humor": 47, "Curiosity": 62 },
     "character": { "tone_casual": 72, "energy_level": 61, "humor_level": 44 }
   }
   ```
6. **스트리밍 표시:** 응답 텍스트 SSE 스트리밍으로 수신, 타이핑 효과로 실시간 렌더링
7. **에러 처리:**

   | 케이스 | 처리 |
   |--------|------|
   | GPT-4o timeout | 3회 재시도 → 에러 메시지 반환 |
   | 응답 파싱 실패 | 텍스트만 반환, 점수 이전 값 유지 |
   | Supabase 저장 실패 | 응답은 정상 반환, 서버 로그만 |
   | GPT-4o 비용 임계치 초과 | GPT-4o-mini로 자동 fallback |

---

### **[AI 피드백] 회화 교정 및 피드백**

- **영어 학습자로서, 내 발화에 대한 자연스러운 교정과 연습 가이드를 받고 싶다. 그래야 대화 흐름을 방해받지 않으면서 실력을 키울 수 있기 때문이다.**

기능 상세 설명

1. **API 엔드포인트:** `POST /api/feedback`
2. **처리 흐름:**
   ```
   1. analyze_utterance()        → 5축 raw 점수
   2. apply_ema()                → 누적 5축 점수
   3. compute_character()        → 캐릭터 파라미터
   4. Gemini 2.5 Flash 호출      → 피드백 JSON 3종
   5. Google TTS 호출            → correction 문장 MP3 변환
   6. 응답 반환: axes + character + feedback + tts_audio
   ```
3. **피드백 3종 구성:**

   | 필드 | 설명 | 예시 |
   |------|------|------|
   | correction | 자연스러운 대안 표현 / 교정 | "You could say 'What's up?' instead." |
   | tone_feedback | 말투·에너지·유머에 대한 격려 | "Your casual tone is really coming through!" |
   | practice_prompt | 이어서 연습할 질문 제시 | "Can you rephrase that more formally?" |

4. **피드백 타이밍:** 대화 흐름을 방해하지 않도록 발화 직후가 아닌 별도 피드백 패널로 표시. 세션 종료 시 요약 피드백 제공 (Post-MVP)
5. **TTS 연동:** correction 문장을 Google TTS로 변환해 사용자가 올바른 발음을 들을 수 있도록 함
6. **Gemini 실패 시 fallback:**
   ```json
   {
     "correction": "Great attempt! Try to vary your sentence structure for more natural flow.",
     "tone_feedback": "Your expression is coming through clearly — keep it up!",
     "practice_prompt": "Can you try expressing the same idea in a different way?"
   }
   ```

### **[STT] 음성 입력 (Speech-to-Text)**

- **영어 학습자로서, 타이핑 없이 말로 영어 문장을 입력하고 싶다. 그래야 실제 회화처럼 자연스럽게 연습할 수 있기 때문이다.**

기능 상세 설명

1. **API 엔드포인트:** `POST /api/stt`
2. **기술:** Google Cloud Speech-to-Text v1
3. **입력 처리 흐름:**
   - 사용자가 채팅창 마이크 버튼 클릭 → MediaRecorder 녹음 시작
   - 녹음 완료(버튼 재클릭 또는 자동 감지) → webm/opus 파일 생성
   - multipart/form-data로 `/api/stt` 전송
   - 서버에서 Google STT API 호출
   - 응답 `{ transcript, confidence }` 수신
   - transcript를 입력창에 자동 삽입
4. **지원 오디오 포맷:** webm/opus (브라우저 MediaRecorder 기본값), MP3, WAV, FLAC
5. **설정값:**
   - 언어: `en-US`
   - 모델: `latest_long`
   - 자동 구두점: 활성화
   - 최대 인식 길이: 60초 (그 이상은 별도 처리 필요)
6. **실패 처리:**
   - 무음 입력 시: `{ transcript: "", confidence: 0.0 }` 반환
   - Google STT 오류 시: 502 에러 → UI에서 "다시 말씀해주세요" 안내

### **[TTS] 음성 출력 (Text-to-Speech)**

- **영어 학습자로서, AI 캐릭터의 응답과 교정 문장을 음성으로 들을 수 있으면 좋겠다. 그래야 올바른 발음을 귀로 확인할 수 있기 때문이다.**

기능 상세 설명

1. **API 엔드포인트:** `POST /api/tts`
2. **기술:** Google Cloud Text-to-Speech v1
3. **사용 시점:**
   - AI 피드백의 correction 문장 자동 읽기
   - 사용자 요청 시 AI 응답 전체 읽기 (선택적)
4. **요청 형식:**
   ```json
   { "text": "You could say 'What's up?' instead.", "voice": "en-US-Journey-F", "speaking_rate": 1.0 }
   ```
5. **응답 형식:**
   ```json
   { "audio_b64": "...(base64 MP3)...", "voice": "en-US-Journey-F", "encoding": "MP3" }
   ```
6. **설정값:**
   - 기본 음성: `en-US-Journey-F` (자연스러운 여성 미국 영어)
   - 오디오 인코딩: MP3
   - 이어폰 최적화 효과: `headphone-class-device` 적용
7. **FE 재생:** `audio_b64`를 `<audio>` 태그 또는 Web Audio API로 자동 재생

## 4. 캐릭터 시각화

### **[캐릭터 시각화] Canvas2D Superformula 실시간 캐릭터**

- **영어 학습자로서, 내 발화 스타일 변화가 시각적으로 표현되는 캐릭터를 보고 싶다. 그래야 내 말투가 AI에게 영향을 주고 있다는 것을 직관적으로 느낄 수 있기 때문이다.**

기능 상세 설명

1. **기술:** HTML5 Canvas2D + Superformula 수식
2. **렌더링:** `requestAnimationFrame` 기반 60fps 루프
3. **5축 → Canvas 파라미터 매핑:**

   | 캐릭터 파라미터 | Canvas2D 변화 | 구체적 동작 |
   |----------------|---------------|-------------|
   | tone_casual | 꼭짓점 수 (m 파라미터) | 낮을수록 각진 별, 높을수록 둥글고 유기적인 형태 |
   | Formality | 각진 정도 (n1 파라미터) | 높을수록 기하학적, 낮을수록 부드럽게 |
   | energy_level | 캐릭터 크기 / 눈 크기 | 높을수록 크고 반짝이는 눈, 낮을수록 작고 차분한 눈 |
   | Intimacy | 색상 (HSL hue) | 낮을수록 차가운 파랑, 높을수록 따뜻한 핑크 |
   | humor_level | 색상 채도 / 불규칙성 | 높을수록 채도 높고 장난스러운 형태 |
   | Curiosity | 호흡 효과 (scale sin 주기) | 높을수록 리드미컬한 부유 효과 강도 증가 |

4. **전환 효과:**
   - lerp(선형 보간) 0.3초 부드러운 형태 전환
   - 눈 깜빡임: 3~6초 랜덤 인터벌
   - Idle 부유 효과: `y += sin(time)` 기반 숨쉬기 애니메이션

5. **업데이트 시점:** AI 응답 도착 직후 → 새 캐릭터 파라미터 값으로 부드럽게 전환
6. **로딩 상태:** 응답 생성 중 캐릭터 회전/진동 애니메이션 표시
7. **성능 최적화:** 파라미터 변화 없으면 렌더링 스킵 (dirty flag)

8. **데모 시나리오별 캐릭터 변화 예시:**

   | 발화 스타일 | 예상 형태 | 색상 |
   |-------------|-----------|------|
   | 캐주얼 ("yo lol") | 둥근 별, 큰 눈 | 핑크/따뜻한 계열 |
   | 격식 ("I would like to...") | 각진 사각형, 작은 눈 | 차가운 파랑 |
   | 유머 ("omg have you seen that meme") | 불규칙한 별, 채도 높음 | 선명한 색상 |
   | 탐구형 ("Why do you think...") | 호흡 효과 강함, 균형잡힌 형태 | 중간 채도 |

## 5. 5축 패널

### **[5축 패널] 발화 스타일 수치 시각화**

- **영어 학습자로서, 내 발화 스타일이 5개 축으로 수치화된 것을 실시간으로 보고 싶다. 그래야 내 대화 패턴이 어떻게 변화하는지 파악할 수 있기 때문이다.**

기능 상세 설명

1. **위치:** 채팅 화면 우측 패널
2. **표시 내용:**
   - 5개 축 각각: 축 이름 + 수치(0~100) + 바(Bar) 형태 표시
   - 현재 캐릭터 파라미터 3종: tone_casual, energy_level, humor_level
3. **업데이트 시점:** 메시지 전송 후 AI 응답 수신 시 애니메이션으로 수치 변화
4. **수치 변화 애니메이션:** 이전 값 → 새 값으로 0.3초 부드러운 숫자/바 전환
5. **디버그 모드 (선택):** tone_casual, energy_level, humor_level 수치 토글 표시
6. **시각적 기준:**
   - 바 색상: 수치에 따라 그라데이션 (낮음=차가운 색, 높음=따뜻한 색)

## 6. 세션 관리

### **[세션 관리] 세션 상태 및 토큰 관리**

- **영어 학습자로서, 대화 세션 내에서 맥락이 유지되고 남은 대화 횟수를 확인하고 싶다. 그래야 대화 흐름이 끊기지 않고 언제 충전이 필요한지 미리 알 수 있기 때문이다.**

기능 상세 설명

1. **세션 생성:** 온보딩 완료 시 UUID 기반 `session_id` 자동 생성
2. **세션 저장:** Supabase `sessions` 테이블 + 브라우저 `localStorage`:
   ```json
   {
     "session_id": "UUID",
     "user_id": "UUID",
     "english_level": "B1",
     "axes": { "Formality": 50, "Energy": 50, "Intimacy": 40, "Humor": 30, "Curiosity": 80 },
     "tokens_remaining": 30
   }
   ```
3. **DB 저장 (Supabase):**
   ```sql
   sessions   (id UUID, user_id, english_level, created_at)
   messages   (id, session_id, role, content, axes JSONB, character JSONB, created_at)
   user_tokens (user_id, tokens_remaining, last_recharged_at)
   ```
4. **대화 맥락 주입:** 현재 세션의 최근 N턴(약 10턴)을 GPT 시스템 프롬프트 history에 직접 주입
5. **토큰 차감:** 메시지 1턴(사용자 발화 1회 + AI 응답 1회) 당 토큰 1개 차감
6. **토큰 만료 처리:** 아래 [결제] 섹션 참고

## 7. 결제

### **[결제] 유료 플랜 및 토큰 구매**

- **영어 학습자로서, 대화 토큰을 구매하여 제한 없이 Pally와 대화하고 싶다. 그래야 학습 흐름이 끊기지 않고 원하는 만큼 연습할 수 있기 때문이다.**

기능 상세 설명

#### 토큰 기반 과금 구조

1. **토큰 정의:** 대화 1턴(사용자 메시지 1회 발신 + AI 응답 1회 수신) = 토큰 1개 소모
2. **신규 가입 시 무료 토큰 지급:** 가입 완료 후 무료 체험 토큰 N개 자동 지급 (수량 추후 확정)
3. **토큰 충전 방식 (패키지 구매):**

   | 패키지 | 토큰 수 | 가격 | 단가 |
   |--------|---------|------|------|
   | 스타터 | 30턴 | ₩2,900 | ₩97/턴 |
   | 베이직 | 100턴 | ₩7,900 | ₩79/턴 |
   | 플러스 | 300턴 | ₩19,900 | ₩66/턴 |
   | 무제한 (월정액) | 무제한 | ₩9,900/월 | — |

   > 가격 및 토큰 수량은 출시 전 확정 예정

4. **결제 수단:** 카드, 카카오페이, 네이버페이 (PG사 연동)
5. **구독 관리:** 월정액 자동 갱신, 언제든 해지 가능

---

#### 토큰 만료 처리 (핵심 UX)

- **영어 학습자로서, 토큰이 소진되면 즉시 안내를 받고 결제 창으로 이동하고 싶다. 그래야 대화가 갑자기 중단되는 당황스러운 경험을 피할 수 있기 때문이다.**

기능 상세 설명

1. **잔여 토큰 표시:** 채팅 화면 상단에 남은 토큰 수 상시 표시 (예: "💬 남은 대화: 5턴")
2. **토큰 임박 경고:** 잔여 토큰 3개 이하 시 토스트 알림 표시
   - 예: "대화가 3번 남았어요. 충전하면 계속할 수 있어요!"
3. **토큰 소진 시 처리 흐름:**
   ```
   사용자 메시지 전송
   → 서버에서 토큰 잔액 확인
   → 잔액 0개: 메시지 처리 중단
   → 대화 화면 위에 결제 유도 바텀시트(Bottom Sheet) 표시
   → 사용자가 충전하거나 닫을 때까지 입력창 비활성화
   ```
4. **토큰 소진 바텀시트 구성:**
   - 상단: 캐릭터 이미지 + "{pally_name}(이)가 더 대화하고 싶어해요!"
   - 중단: 토큰 패키지 선택 (스타터 / 베이직 / 플러스)
   - 하단: "충전하기" 버튼 (결제 플로우 진입) + "나중에" 링크 (바텀시트 닫기)
5. **충전 후 복귀:** 결제 완료 시 자동으로 채팅 화면으로 복귀, 이전 대화 맥락 유지하여 즉시 재개
6. **"나중에" 선택 시:** 바텀시트 닫힘, 채팅 입력창은 비활성화 상태 유지. 입력창 탭 시 충전 유도 배너 재표시

---

#### 결제 흐름

1. 패키지 선택 → 결제 수단 선택 → PG사 결제창 → 완료 확인
2. 결제 완료 시: `user_tokens.tokens_remaining` += 구매 토큰 수
3. 결제 실패 시: 실패 안내 토스트 + 결제 재시도 유도
4. 결제 이력: `payment_history` 테이블에 저장 (user_id, amount, tokens, created_at)

---

#### 장기 페르소나 및 계정 관리 *(Post-MVP)*

- **장기 페르소나 저장:**
   ```sql
   user_persona (user_id, F, E, I, H, C_ema, updated_at)
   ```
   세션 종료마다 EMA로 5축 누적 업데이트
- **단기 맥락 vs 장기 페르소나 분리:**
  - 단기: `session_messages` (최근 10턴, 세션 종료 후 벡터화)
  - 장기: `user_persona` (EMA 누적 5축)
  - 장기 기억 검색: `conversation_embeddings` (pgvector RAG)
- **장기 미접속 복귀:** 마지막 누적 5축 값으로 캐릭터 복원 + 복귀 인사 메시지
- **데이터 삭제 권리:** 대화 기록 삭제/수정 요청 시 DB 완전 삭제 가능

## 9. ML 발화 분석기 *(Post-MVP)*

### **[ML 발화 분석기] 딥러닝 기반 5축 분석 고도화**

- **영어 학습자로서, 더 정확하게 내 발화 스타일이 분석되길 원한다. 그래야 미묘한 말투 변화도 캐릭터에 반영될 수 있기 때문이다.**

기능 상세 설명

1. **기술:** `sentence-transformers` 임베딩 + 5축 regression head
2. **학습 데이터:** Rule-Based 분석기 출력 쌍 자동 축적 (앱 사용 데이터)
3. **평가 지표:** 각 축별 MAE (Mean Absolute Error)
4. **서빙 구조:**
   - Feature flag로 Rule-Based ↔ ML 라우팅 전환
   - API: `POST /api/analyze/ml`
5. **A/B 비교 실험:**
   - 그룹 A: ML 기반 분석
   - 그룹 B: Rule-Based 분석 (fallback 유지)
   - 비교 지표: MAE per axis, 세션 지속 시간, 재방문율
6. **최소 데이터 임계:** 앱 누적 사용 데이터 1,000쌍 이상 확보 후 학습 착수

4. **슬랭 필터링:**

## 11. 안전장치 *(Post-MVP)*

### **[안전장치] 과몰입 방지 및 AI 윤리 설계**

- **영어 학습자로서, 건강한 방식으로 AI를 학습 도구로만 활용하고 싶다. 그래야 과몰입이나 정서적 의존 없이 지속 가능하게 공부할 수 있기 때문이다.**

기능 상세 설명

1. **학습 도구 포지셔닝 유지:**
   - 캐릭터는 "영어 학습 파트너"로 일관 프레이밍
   - 로맨틱 관계, 감정 지지 방향 대화: 시스템 프롬프트 레벨에서 차단
   - 캐릭터가 "나 진짜 사람이야" 류 응답 생성 금지 (시스템 프롬프트 명시)

2. **사용 패턴 피드백:**
   - 주간 사용 리포트 제공: "이번 주 N회 대화, 총 M분"
   - 일 사용 시간 임계값(2시간) 초과 시 "잠깐 쉬어가세요" 넛지 표시
   - "현실 친구와도 영어 연습해보세요" 류의 현실 연결 넛지

3. **AI 명시 표시:**
   - 채팅 UI 상단 상시 표시: "AI 캐릭터와 대화 중"
   - 온보딩에서 AI 파트너임을 명확히 안내

4. **사용자 데이터 제어권:**
   - 특정 대화 내용 삭제 요청 시 DB에서 완전 삭제
   - 개인화 데이터 초기화 기능 (5축 누적 점수 리셋)
   - Prompt injection 방어: 사용자 입력값 시스템 프롬프트와 분리 처리

## 데모 시나리오 (June 7)

**시나리오 1 — 캐주얼 발화**
```
입력: "yo what's up lol, u wanna hang or nah?"
예상: Formality↓, Energy↑, Humor↑
캐릭터: 둥근 별, 핑크, 큰 눈
응답: 가볍고 장난스러운 슬랭 포함 응답
```

**시나리오 2 — 격식 발화**
```
입력: "I would like to formally inquire about this matter."
예상: Formality↑, Energy↓, Humor↓
캐릭터: 각진 형태, 차가운 파랑, 작은 눈
응답: 정중하고 격식 있는 응답
```

**시나리오 3 — 연속 대화로 성격 변화**
```
격식체로 시작 → 점점 캐주얼하게 바꾸기
→ 5축 패널 수치 변화 + 캐릭터 형태 서서히 전환을 실시간으로 시연
```
