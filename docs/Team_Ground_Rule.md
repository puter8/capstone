## 1. 팀의 기본 가치

우리 팀은 다음의 가치를 기반으로 프로젝트를 수행한다.

- 책임감: 맡은 역할은 끝까지 책임지고 수행한다
- 존중: 팀원의 의견을 존중하고 경청한다
- 소통: 문제를 숨기지 않고 공유한다
- 협력: 개인보다 팀의 성과를 우선한다

---

## 2. 역할 분담 규칙

팀 프로젝트의 효율적인 진행을 위해 역할을 명확히 한다.

- PM: 전체 일정 관리 및 스프린트 계획, 요구사항 정의 및 문서화, 팀 커뮤니케이션 조율, 발표 및 데모 준비
- Frontend: UI/UX 설계 및 구현, 사용자 인터페이스 개발, API 연동 및 상태 관리, 반응형 디자인 및 접근성 처리
- Backend: 서버 및 API 설계/구현, DB 설계 및 관리, 인증/인가 처리, 인프라 배포 및 운영
- AI: AI 모델 선정 및 파인튜닝, 데이터 수집·전처리·분석, 모델 성능 평가 및 개선, AI 파이프라인 설계 및 Backend 연동

추가 규칙

- 모든 팀원은 최소 1개 이상의 핵심 역할을 담당한다
- 역할 변경이 필요할 경우 팀 회의를 통해 결정한다
- 특정 팀원에게 업무가 과도하게 집중되지 않도록 조정한다

---

## 3. 회의 규칙

효율적인 협업을 위해 다음과 같은 회의 규칙을 따른다.

- 필요 시 추가 회의 진행
- 회의 전 안건 공유
- 회의 후 회의록 작성

추가 규칙

- 회의 지각 시 사전 연락 필수

---

## 4. 커뮤니케이션 규칙

팀원 간 원활한 소통을 위해 다음을 준수한다.

- 주요 소통 채널: 카카오톡 단체 채팅방
- 중요 공지는 단체 채팅방에 공유
- 메시지 응답은 ASAP

추가 규칙

- 일정 변경은 반드시 팀 전체에게 공유
- 개인적으로 결정하지 않고 팀 논의를 거친다

---

## 5. 작업 및 일정 관리 규칙

프로젝트 진행을 체계적으로 관리한다.

- 모든 작업 일정은 공유 문서에 기록
- 마감 최소 2일 전에 중간 점검 진행
- 작업 결과는 팀원들과 공유

개발 프로젝트의 경우

- 코드 관리는 GitHub 사용
- 주요 기능 업데이트 시 팀원에게 설명

---

## 6. 갈등 해결 규칙

팀 프로젝트에서는 갈등이 발생할 수 있다. 우리 팀은 다음 방식으로 해결한다.

- 문제가 발생하면 즉시 팀 회의에서 논의
- 개인 비난이 아닌 문제 해결 중심으로 대화
- 해결되지 않을 경우 지도교수 또는 멘토 상담

---

## 7. 참여 규칙

모든 팀원은 다음을 준수한다.

- 팀 활동에 성실히 참여한다
- 무단 불참을 하지 않는다
- 프로젝트 일정 준수

지속적인 참여 부족 시

- 팀 내부 경고
- 반복될 경우 교수님에게 보고

---
