## 문제 정의

글로벌 영어 학습 앱의 첫 주 이탈률은 60%를 상회하며, 약 44%의 학습자가 초기 단계에서 이탈한다.  
근본 원인은 기존 서비스가 **고정된 응답 패턴**에 머물러, 사용자의 성향과 말투를 전혀 반영하지 못하기 때문이다.  
또한 MZ세대 중심의 슬랭·밈이 실제 회화에서 차지하는 비중이 크지만, 대부분의 서비스는 이를 빠르게 반영하지 못하고 있다.

---

## 해결책

> "내가 대화할수록, AI가 나를 닮아간다"

### 1. CHARACTER MATRIX 시스템

사용자 발화를 5가지 성격 축(Formality, Energy, Intimacy, Humor Style, Curiosity)으로 수치화하고 LLM 시스템 프롬프트에 실시간 반영하여
AI의 말투·슬랭 빈도·리액션 강도를 동적으로 조절한다.

### 2. 슬랭 자동 수집 파이프라인

Reddit PRAW로 데이터를 수집하고, GitHub Actions로 주기적으로 실행하여
Supabase pgvector로 벡터 검색을 수행한다. 외부 벡터 DB 없이 기존 DB 내에서 RAG를 구현하는 절충안이다.

### 3. Rive 캐릭터 시각화

MATRIX 수치 변화에 따라 캐릭터의 표정·동작·색상이 실시간으로 달라지는 인터랙티브 UI를 구현하여  
사용자가 자신만의 AI 파트너를 키워가는 경험을 제공한다.

## 기술 선택 이유

| 기술           | 선택 이유                                                   |
| -------------- | ----------------------------------------------------------- |
| FastAPI        | Python 기반 단일 백엔드, 비동기 처리로 LLM 응답에 최적화   |
| Supabase       | Auth + RLS + PostgreSQL 통합, 프론트엔드 직접 CRUD 가능     |
| Rive           | 실시간 파라미터 연동 애니메이션, 웹 경량화에 강점           |
| GitHub Actions | 별도 인프라 없이 슬랭 수집 파이프라인 스케줄링 가능         |
