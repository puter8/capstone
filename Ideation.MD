# 💡 Project Ideation: 소설 보조 AI 어시스턴트

## 1. 페르소나 (Target Audience)
* **메인 타겟**: 머릿속에 스토리는 가득하지만, 이를 뒷받침할 어휘나 문장력이 부족해 글쓰기를 망설이는 예비 작가.
* **서브 타겟**: 보고서나 일상적인 글쓰기 외에 좀 더 감성적이고 풍부한 표현을 익히고 싶은 일반인.

## 2. 문제 정의 (Problem Statement)
* **어휘의 한계**: 평소 사용하는 단어의 범위가 좁아 문장이 단조롭고 진부하게 느껴짐.
* **묘사의 어려움**: 감정이나 풍경을 구체적으로 설명하고 싶지만 적절한 비유나 형용사를 떠올리기 힘듦.
* **범용 AI의 한계**: 기존 ChatGPT 등은 너무 정보 전달 중심이거나 문체가 기계적이라 소설 특유의 '맛'을 살리기 부족함.

## 3. 해결 방안 (Solution)
* **문학 특화 데이터 학습**: 소설적 허용과 비유, 은유가 가득한 텍스트 데이터를 집중적으로 파인튜닝하여 '문학적 감수성'을 가진 모델 구축.
* **어휘 학습 루프**: 단순히 고쳐주는 것을 넘어, 사용자가 자주 틀리거나 부족한 어휘를 학습할 수 있도록 유도하는 어시스턴트 기능.

## 4. 핵심 가치 (Core Value)
1. **창작의 장벽 완화**: 누구나 소설가가 될 수 있는 문턱을 낮춤.
2. **언어의 풍성함**: 한국어 특유의 아름다운 의성어, 의태어 및 고어 활용 극대화.
3. **개인 맞춤형 문체**: 파인튜닝을 통해 작가 개개인의 스타일을 유지하면서도 발전된 형태의 문장 제안.

## 5. 향후 발전 가능성 (Roadmap)
* **장르별 모드**: 로맨스, 추리, 판타지 등 장르에 최적화된 어휘 사전 선택 기능.
* **협업 필터링**: 내가 쓴 문장과 유사한 유명 작가의 문체를 분석해 주는 기능.
* **플러그인 확장**: 웹소설 연재 플랫폼이나 워드 프로세서 내에 플러그인 형태로 삽입.
