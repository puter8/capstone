## Design System

| Property | Value |
|----------|-------|
| Tool | none (no shadcn — manual Tailwind + custom components) |
| Preset | not applicable |
| Component library | none (hand-rolled React + Tailwind; `frontend/components/{audio,chat,nav,ui}/`) |
| Icon library | inline JSX SVG (extracted from Figma `mvp design` nodes — no `lucide-react` / `heroicons` install in 1A) |

**Why no shadcn:** This phase is mock-only frontend on a one-screen mobile chat shell. Adding a Radix-backed primitives layer would be scope creep against the "간단하게" mandate in CONTEXT.md and offers no benefit for the 4 component families this phase needs (TalkButton, ChatBubble, HistorySheet/Row, BottomNav, Toast). The Figma file is the design system; DESIGN.md and `tailwind.config.ts` already mirror it.

## Typography

Source: `frontend/tailwind.config.ts` lines 34-50 (already implemented Phase 0) + `DESIGN.md § Typography`.

**Pretendard Variable** is the only font family. `letterSpacing: 0` across all tokens. All sizes in `px`. All line-heights in `px` (not unitless ratios) — matches Figma exactly.

The full 15-token set is locked at the project level (`tailwind.config.ts`). Phase 1A uses the **subset below** — do not introduce new sizes or weights inside `components/`.

### Phase 1A active subset (5 sizes, 2 weights)

| Role | Tailwind class | Size | Line | Weight | Where used in 1A |
|------|----------------|------|------|--------|------------------|
| Display | `text-title-1` | 24px | 36px | 600 SemiBold | Empty-state heading "오늘은 어떤 이야기를 해볼까요?" |
| Heading | `text-subtitle-sb` | 18px | 24px | 600 SemiBold | Chat-bubble Pally reply emphasized text, history sheet section title |
| Body | `text-body` | 16px | 24px | 400 Regular | Chat-bubble user transcript + Pally reply default, empty-state subtitle, toast message body |
| Caption SB | `text-body-2-sb` (14px/24/600) **OR** `text-caption-1` (12px/16/400) — see note | 12-14px | 16-24px | varies | Sender labels "YOU" / "Pally" — see note below |
| Caption | `text-caption-1` | 12px | 16px | 400 Regular | Timestamps, toast inline error text, mic permission-denied inline message |
| Smallest | `text-caption-2` | 11px | 12px | 400 Regular | BottomNav tab labels "홈 / 히스토리 / 새 대화 / 랭킹 / 내 정보" |

**Note on sender labels (YOU / Pally):** `DESIGN.md § Known limitations #1` documents that Figma currently renders these at Title 20-24px due to Pretendard not being installed in the Figma desktop environment. The **design intent** (per the Decisions Log) is `text-caption-1` (12px SemiBold). For 1A implementation, **use `text-body-2-sb` (14px SemiBold)** as the pragmatic mid-point: it is unambiguously below the body it labels, semibold for hierarchy, and matches the project's existing 600-weight token set without inventing a 12px-SemiBold combination. When Figma is re-synced after Pretendard install (Senior plan B4), this file and the implementation should both move to 12px/SemiBold. Until then, **`text-body-2-sb` is the locked choice**.

### Weight rule

Only two weights ship in 1A: **400 (Regular)** and **600 (SemiBold)**. Do not import or use other Pretendard weights. The tokens above already bake the weight in — never layer `font-bold` or `font-medium` on top.

### Line-height rule

Token tokens have line-height baked in. Do not override with `leading-*` utilities. Body lines are 24px on a 16px font (1.5 ratio), headings 36px on 24px (1.5 ratio) — consistent with the recommended 1.5 body / 1.2 heading floor.

### Accent reserved-for list (explicit)

`primary` orange `#fe9012` is reserved for, and **only** for, the following elements in Phase 1A:

1. TalkButton idle disc fill (the mic-icon-bearing circle).
2. TalkButton idle outer ring at 18% opacity.
3. "Pally" sender-label text (the 14px SemiBold tag preceding Pally's reply).
4. The active BottomNav tab's icon stroke + label text. (All 5 tabs are visually disabled in 1A per CONTEXT D-17, so this state is never reached at runtime, but the rule stands for any future activation.)
5. History view bottom decorative stripe (13px, but using `primary-soft` `#ffb84a`, not full `primary`).

`primary` orange must **not** be used for: body text, the back-button stroke, default icon strokes, border decorations, or hover states on disabled tabs. Any future addition requires a DESIGN.md decisions-log entry.

`accent` teal `#00c3d0` is reserved for, and **only** for: the "YOU" sender-label text. No other usage in 1A.

`error` red `#ef4444` is reserved for: the recording-active disc + pulse ring (CONTEXT D-12 states `recording` / `processing` / `speaking` share this visual) and the inline "다시 한 번 말해주세요" error message text. No other usage.

### Contrast caveat (deferred audit)

`primary` `#fe9012` on `surface-raised` `#ffffff` measures ~2.84:1 — below WCAG AA 4.5:1 floor for normal-sized body text. In Phase 1A this is acceptable because:

- The orange is used **only** on large surfaces (the 96px disc and 13px stripe) where AA-Large (3:1) applies and passes.
- The single small-text use is the "Pally" sender label (14px SemiBold), which Senior plan A6 explicitly defers to Phase B for a darker fallback (`#e67e00`) audit.

**Lock for 1A:** Use the Figma colors as-is. Do not pre-emptively darken. The Phase B audit will decide the production sender-label color. If the gsd-ui-checker flags this, the rationale lives here and in DESIGN.md decisions-log entry 2026-05-25 "Senior polish 1차 적용 완료".

## Interaction Timing

Magic numbers are not magic — each has a documented rationale.

| Timing | Value | Source | Why |
|--------|-------|--------|-----|
| Mock latency | **800ms** | CONTEXT D-04 | Approximates plausible Phase-1C end-to-end (STT + Gemini + TTS) round-trip so the UX shell rehearses real waiting. Fixed (no jitter) so demo screen recordings are deterministic. |
| Speaking-state hold | **1500ms** | CONTEXT D-10, D-13 | Phase 1A does not play TTS; the speaking state simulates the period when audio *would* be playing. 1.5s is long enough for the user to perceive "Pally responded" without lingering. Phase 2 replaces with real `<audio>` playback duration. |
| Max recording duration | **30000ms (30s)** | CONTEXT D-08 | `MediaRecorder.stop()` auto-triggered at 30s. Floor on `MediaRecorder` capacity is much higher; 30s caps abuse + mirrors what a single conversational turn realistically needs. |
| Toast auto-dismiss | **4000ms (4s)** | Discretion (this spec) | Long enough to read 1 Korean line at calm pace, short enough to clear before the user re-attempts. Tap-to-dismiss also available. |
| Chat-bubble enter motion | 200ms ease-out, fade + slide-up 8px | DESIGN.md § Motion | Standard short-enter token. |
| Mic press feedback | scale 0.95, 150ms | DESIGN.md § Motion | Standard tactile press. |
| Reducer state transitions | synchronous (no timeout) | RESEARCH §Pattern 2 | All recorder transitions dispatched from event handlers / promise callbacks. No setTimeout-based "after N ms move to state X" except the speaking → idle 1500ms timer above. |

## Anti-Patterns to Reject in Review

The gsd-ui-checker and code-review pass MUST flag any of the following:

1. **Arbitrary spacing in JSX** — `p-[13px]`, `mt-[7px]`, `gap-[10px]`. Use the 4-grid Tailwind utilities. Exception: the 132px / 144px Figma decorative-ring sizes if they ever appear outside the TalkButton are flagged (they are TalkButton-internal).
2. **Inline raw hex colors in JSX** — `style={{color: '#fe9012'}}` or `className="bg-[#fcf9f6]"`. The Phase 1A plan adds these to `tailwind.config.ts theme.extend.colors`; executor uses `bg-primary`, `text-accent`, etc.
3. **New color tokens introduced without a DESIGN.md decisions-log entry.** If a state needs a color outside the table above, stop and update DESIGN.md first.
4. **New font sizes / weights outside the 15-token set.** No `text-[17px]`, no `font-medium`. The two-weight rule (400 / 600) is non-negotiable for 1A.
5. **`opacity-50` or similar dimming on disabled BottomNav tabs.** DESIGN.md mandates Figma-as-is — full opacity + `aria-disabled="true"` + no onClick + `cursor-default` (RESEARCH §Anti-Patterns).
6. **Animations that 1A cannot deliver** — Pally morph, typewriter reveal, audio waveform, splash fade. All deferred. Plain CSS transitions on chevron rotation and history sheet slide are OK.
7. **Magic numbers without a comment.** The 800ms / 1500ms / 30000ms / 4000ms each must carry a short `// 1A mock latency per CONTEXT D-04` comment at the call site for downstream reviewers.
8. **Locking the implementation to 360px** — viewport target is **402px** per CONTEXT.md and commit `a2c2bb4`. Other viewport widths are out-of-scope (no media queries beyond what is in the base Tailwind reset).
9. **Reading `hint_ko` from the mock response in UI.** Phase 2 surfaces it; 1A does not. The mock fixture includes the field only for wire-shape parity.
10. **Playing `tts_audio`.** Forbidden in 1A per CONTEXT D-10. No `new Audio()`, no `<audio>` tags.
